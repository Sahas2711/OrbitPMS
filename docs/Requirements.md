Task 2 – Hotel / PMS Mini Module
Objective
Your next task is to build a mini Hotel Management / PMS (Property Management System) application.
The goal of this project is to understand how real-world hospitality software works and how multiple modules are connected in a business application.
This project should be developed with proper frontend + backend structure and clean UI.
Modules Required
1. Authentication
Implement:
User Login
User Registration
Logout functionality
Optional:
Role-based login (Admin / Staff)
2. Room Management
Create a module to manage hotel rooms.
Features:
Add room
Edit room
Delete room
Room number
Room type (Standard, Deluxe, Suite)
Room price
Room status:
Available
Occupied
Maintenance
3. Booking System
Implement room booking functionality.
Features:
Create booking
Guest name
Check-in date
Check-out date
Number of guests
Assign room
Booking status
Validation:
Prevent double booking for same room and date
4. Check-In / Check-Out Module
Features:
Check-in guest
Check-out guest
Update room status automatically
Calculate total stay amount
5. Invoice Generation
Create a simple invoice page.
Invoice should contain:
Guest name
Room details
Stay duration
Room charges
Total amount
Optional:
Download invoice as PDF
6. Availability Calendar
Display room availability.
Features:
Show booked dates
Show available rooms
Calendar-based UI preferred
Technical Requirements
Frontend
Use:
React.js
Tailwind CSS / Bootstrap
Focus on:
Responsive design
Proper UI alignment
Reusable components

Use:
Fast API

Create proper relationships between:

Invoices
Use clean database schema and naming conventions.
